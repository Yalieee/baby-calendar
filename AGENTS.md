# Agent 指引

## 專案定位

`baby-calendar` 是一個只部署成 static page 的新生兒成長照護筆記網站。目標是追蹤新生兒不同階段需要注意的事項，並整理照顧過程中可能遇到的問題、警訊與處理方向。

## 開發原則

- 維持純前端靜態網站，不加入後端服務、資料庫或需要伺服器執行的功能。
- 內容應以照顧者容易理解為優先，分類清楚，避免過度技術化。
- 新增功能時優先使用專案現有的 Vue 與 Vite 結構，並跟隨目前程式碼風格。
- 醫療相關內容應以提醒、觀察與就醫警訊為主，不取代專業醫療建議。
- 支援手機介面，所有頁面與互動都應適合手機閱讀與操作。

## 靜態網站原則

網站預期由 GitHub Pages 提供靜態檔案，不需要後端服務。開發時請避免加入會讓專案依賴伺服器端功能的設計，例如：

- 後端 API、資料庫或伺服器端儲存。
- 需要登入、帳號系統或伺服器工作排程的功能。
- 會把使用者資料送到後端服務處理的表單或整合。

內容與互動應盡量在前端完成，並由 Vite 打包成 GitHub Pages 可提供的靜態檔案。版面應採用 responsive 設計，優先確保手機螢幕上的可讀性與易用性。
